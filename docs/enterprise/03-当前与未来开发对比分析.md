# 当前与未来开发对比分析

> 本文档对比 SkillHub 现有功能与 SkillHub-Enterprise 未来规划之间的差距，分析需要新增、修改或增强的功能模块，为实施计划提供依据。
>
> **文档版本：** v1.0
> **创建日期：** 2026-04-19
> **项目名称：** SkillHub-Enterprise（基于 SkillHub v0.x 企业扩展版）

---

## 一、总体对比概览

### 1.1 功能维度对比

| 功能领域 | 现有能力 | 目标能力 | 差距 | 优先级 |

| **RAG 检索** | 无 | 向量存储 + 混合检索 | 全新能力 | P0 |
| **知识图谱** | 无 | 实体识别 + 关系抽取 | 全新能力 | P2 |
| **执行环境** | 无 | Docker 沙箱执行 | 全新能力 | P2 |
| **MCP 协议** | 无 | Agent 工具调用 | 全新能力 | P2 |
| **IDE 插件** | 无 | VS Code / IDEA 集成 | 全新能力 | P3 |
| **即时通讯采集** | 无 | 钉钉/飞书集成 | 全新能力 | P3 |

## 二、功能模块详细对比

### 2.1 技能仓库模块

#### 2.1.1 技能分类

| 能力项 | 现状 | 目标 | 差距分析 |
|--------|------|------|---------|
| 分类维度 | 仅 `labels` 表，支持打标签 | 多维度：人员/项目/职能/层级 | 需要新增 `skill_dimensions` 表 |
| 分类层级 | 无层级，平铺 | 支持父子层级 | 需要 `parent_id` 自关联 |
| 多维度标签 | 一个技能可有多个标签，但不能跨维度 | 一个技能可同时打多个维度的标签 | 需要维度映射表 |
| 分类浏览 | 无专门的分类浏览入口 | 支持按维度筛选浏览 | 需要新增 API 和前端页面 |
| 分类管理 | Admin 可管理标签 | 各维度管理员可管理 | 需要 RBAC 扩展 |

**差距影响：** 中
**实施复杂度：** 中

#### 2.1.2 技能搜索

| 能力项 | 现状 | 目标 | 差距分析 |
|--------|------|------|---------|
| 搜索方式 | PostgreSQL LIKE 模糊匹配 | 语义向量搜索 + 混合搜索 | 需要 pgvector 集成 |
| 搜索质量 | 依赖关键词命中 | 自然语言理解 | 需要 Embedding 模型 |
| 搜索性能 | 单表索引即可 | 需要 IVFFlat 或 HNSW 索引 | 需要性能优化 |
| 排序方式 | 按热度/时间/评分 | 多维度相关度排序 | 需要重排序服务 |
| 搜索建议 | 无 | 自动补全/纠错 | Phase 2 |

**差距影响：** 高（直接影响用户体验）
**实施复杂度：** 中

#### 2.1.3 技能详情

| 能力项 | 现状 | 目标 | 差距分析 |
|--------|------|------|---------|
| 基础信息 | 名称/描述/图标 | 保留 + 新增多维度标签 | 兼容 |
| 版本信息 | 支持语义版本 | 支持 + 蒸馏来源标记 | 新增字段 |
| 相关技能 | 无 | 基于向量相似度推荐 | 新增推荐算法 |
| 技能统计 | 下载/评分/收藏 | 保留 + 知识使用次数 | 新增埋点 |

**实施复杂度：** 低

## 七、总结

### 7.1 核心差距

1. **语义搜索能力缺失** - 当前仅有关键词匹配，无法理解语义
2. **知识采集能力缺失** - 无法从员工工作中被动采集知识
3. **RAG 能力缺失** - 无法为 AI Agent 提供知识检索能力
4. **执行环境缺失** - 技能仅能存储，无法执行

### 7.2 实施建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 语义搜索 | 直接提升现有功能体验 |
| P0 | 主动提交 RAG | 快速产生用户价值 |
| P1 | 多维度分类 | 组织治理基础 |
| P1 | Git 自动分析 | 被动采集核心能力 |
| P2 | 知识图谱 | 深度分析基础 |
| P2 | 执行环境 | 技能价值升华 |

### 7.3 关键成功因素

1. **用户参与度** - 员工愿意主动提交知识
2. **搜索质量** - 语义搜索效果达到预期
3. **Git 集成体验** - 授权和分析流程顺畅
4. **合规审查** - 数据采集符合法规要求
