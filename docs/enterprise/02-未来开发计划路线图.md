---

## 一、项目愿景与目标

### 1.1 愿景声明

**将员工的工作经验和项目知识转化为可复用的 AI 技能资产，实现企业知识资产的数字化管理和传承。**

### 1.2 核心价值

| 价值维度 | 描述 |
|---------|------|
| **知识沉淀** | 将员工个人经验转化为组织资产，避免知识流失 |
| **效率提升** | 快速找到相似问题的解决方案，减少重复工作 |
| **培训加速** | 新员工可通过学习历史技能快速上手 |
| **创新复用** | 优秀的工作模式可跨团队复制推广 |

### 1.3 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 技能覆盖率 | 80% 核心项目有对应技能 | 技能包数量 / 项目数 |
| 采用率 | 30% 月活跃用户使用技能 | DAU / 总用户 |
| 知识提取量 | 人均 5 个知识片段 / 月 | 蒸馏任务完成数 / 员工数 |
| 搜索满意度 | 70% 搜索结果被认为有用 | 用户反馈统计 |

#### 4.2.2 语义搜索（基于 pgvector）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 向量索引创建 | 上传技能时自动生成描述向量 | P0 |
| 语义相似搜索 | 支持自然语言查询相关技能 | P0 |
| 混合搜索 | 结合关键词和语义向量搜索 | P1 |
| 搜索结果排序 | 按相关性/下载量/评分排序 | P1 |

**技术实现：**

```sql
-- 安装 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 为技能描述添加向量字段
ALTER TABLE skills ADD COLUMN embedding vector(1536);

-- 向量索引
CREATE INDEX idx_skills_embedding ON skills USING ivfflat (embedding vector_cosine_ops);

-- 语义搜索函数
CREATE OR REPLACE FUNCTION semantic_search(
    query_text TEXT,
    match_count INT DEFAULT 10
)
RETURNS TABLE(
    skill_id BIGINT,
    similarity DOUBLE PRECISION
)
AS $$
BEGIN
    RETURN QUERY
    SELECT
        s.id,
        1 - (s.embedding <=> embed_text(query_text)) AS similarity
    FROM skills s
    WHERE s.state = 'PUBLISHED'
    ORDER BY s.embedding <=> embed_text(query_text)
    LIMIT match_count;
END;
$$ LANGUAGE plpgsql;
```

**embedding 模型选择：**
| 模型 | 维度 | 特点 | 成本 |
|------|------|------|------|
| emb-0 (MiniMax) | 1024 | 中文优化效果好 | 按 token 计费 |
| bge-large-zh-v1.5 | 1024 | 中文优化 | 可本地部署 |
| m3e-large | 1024 | 中文优化 | 可本地部署 |

**配置：**
```yaml
skillhub:
  embedding:
    provider: minimax  # 国内使用 MiniMax（OpenAI 兼容）
    model: emb-0
    api-key: ${MINIMAX_API_KEY}
    base-url: ${MINIMAX_API_BASE_URL:}  # https://api.minimaxi.com/v1 (中国)
    dimensions: 1024
```

#### 4.2.3 主动提交 RAG

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文档上传 | 支持上传经验文档（Markdown/PDF/DOCX） | P0 |
| 自动分段 | 文档自动分段存储为知识片段 | P0 |
| 向量化存储 | 片段自动生成向量并存储 | P0 |
| 知识检索 API | 为 AI Agent 提供 RAG 检索接口 | P0 |
| 员工知识库 | 员工可查看自己提交的所有知识 | P1 |

**技术实现：**

```sql
-- 知识片段表
CREATE TABLE knowledge_chunks (
    id              BIGINT PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id),
    title           VARCHAR(500),                 -- 片段标题
    content         TEXT NOT NULL,              -- 原始内容
    chunk_index     INTEGER,                     -- 片段在原文档中的顺序
    source_document VARCHAR(500),                -- 原始文档名
    source_type     VARCHAR(50),                -- MARKDOWN/PDF/DOCX/WORD
    file_size       BIGINT,                      -- 文件大小
    embedding       vector(1536),               -- 向量
    metadata        JSONB,                      -- 扩展信息（来源、创建时间等）
    created_at      TIMESTAMP NOT NULL,
    updated_at      TIMESTAMP NOT NULL
);

-- 用户知识库
CREATE TABLE user_knowledge_library (
    id              BIGINT PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id),
    name            VARCHAR(200),                -- 知识库名称
    description     TEXT,
    is_public       BOOLEAN DEFAULT FALSE,      -- 是否公开
    created_at      TIMESTAMP NOT NULL,
    updated_at      TIMESTAMP NOT NULL
);

-- 知识库与片段的关联
CREATE TABLE knowledge_library_chunks (
    library_id       BIGINT NOT NULL REFERENCES user_knowledge_library(id),
    chunk_id         BIGINT NOT NULL REFERENCES knowledge_chunks(id),
    created_at      TIMESTAMP NOT NULL,
    PRIMARY KEY (library_id, chunk_id)
);

-- 索引
CREATE INDEX idx_chunks_user ON knowledge_chunks(user_id);
CREATE INDEX idx_chunks_embedding ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX idx_library_user ON user_knowledge_library(user_id);
```


| 功能 | 描述 | 优先级 |

| Git 仓库连接 | 配置员工的工作 Git 仓库 | P0 |

| 技能摘要生成 | LLM 分析生成技能摘要 | P0 |

**技术实现：**

```sql
-- Git 数据源配置
CREATE TABLE git_data_sources (
    id              BIGINT PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id),
    repo_url        VARCHAR(500) NOT NULL,
    repo_name       VARCHAR(200),
    access_token    VARCHAR(255),               -- 加密存储
    last_sync_at    TIMESTAMP,
    sync_status     VARCHAR(20),                -- SYNCED/ERROR/PENDING
    created_at      TIMESTAMP NOT NULL,
    updated_at      TIMESTAMP NOT NULL
);

-- 蒸馏任务表
CREATE TABLE distillation_tasks (
    id              BIGINT PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES users(id),
    source_type     VARCHAR(50) NOT NULL,         -- GIT/FILE/UPLOAD/IDE
    source_id       BIGINT,                        -- 关联 source 表
    status          VARCHAR(20) NOT NULL,          -- PENDING/PROCESSING/COMPLETED/FAILED
    progress        INTEGER DEFAULT 0,             -- 0-100%
    result_summary  TEXT,                          -- 生成的摘要
    error_message   TEXT,
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP,
    created_at      TIMESTAMP NOT NULL
);
```

**分析流程：**
```
1. 员工配置 Git 仓库（授权）
       │
       ▼
2. 定期同步提交历史
       │
       ▼
3. LLM 分析提交模式
   - 识别问题解决类型的提交
   - 提取决策过程和方案
       │
       ▼
4. 生成知识片段建议
       │
       ▼
5. 员工确认/修改后入库
```
